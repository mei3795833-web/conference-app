# 大会管理系统 - 多用户支持更新

## 时间：2026-06-25 11:50 PDT

## 更新内容

### 多用户权限系统

新增三种用户角色，通过不同密码区分：

| 密码 | 角色 | 权限 |
|------|------|------|
| `admin123` | 管理员 | 编辑大会信息、查看统计数据 |
| `staff123` | 工作人员 | 查看统计数据（只读） |
| `view123` | 查看员 | 只读查看大会信息 |

### 权限控制

#### 管理员 (admin)
- 可以编辑所有大会信息（名称、地点、地址、日期、状态）
- 可以查看数据统计（参会人数、已签到）
- 显示保存按钮，可保存修改

#### 工作人员 (staff)
- 只能查看大会信息（输入框禁用）
- 可以查看数据统计
- 显示"您只有查看权限"提示

#### 查看员 (viewer)
- 只能查看大会信息（输入框禁用）
- 不能查看数据统计面板
- 显示"您没有编辑权限"提示

### 界面更新

- 管理面板顶部显示当前用户名称和角色
- 根据权限自动启用/禁用编辑功能
- 根据权限显示/隐藏数据统计面板
- 根据权限显示/隐藏保存按钮

### 文件修改

#### js/app.js
- 添加 `users` 配置对象（包含三种用户角色）
- 添加 `currentUser` 状态跟踪
- 添加 `hasPermission()` 函数检查权限
- 更新 `adminLogin()` 支持多用户验证
- 更新 `renderAdminPanel()` 根据权限渲染不同界面
- 更新 `renderAdmin()` 使用 `currentUser` 替代 `isAdminLoggedIn`
- 更新 `adminLogout()` 清除 `currentUser`

## 使用说明

1. 打开网页点击底部"管理"标签
2. 输入密码：
   - `admin123` - 管理员权限
   - `staff123` - 工作人员权限  
   - `view123` - 只读权限
3. 根据角色权限查看/编辑大会信息
4. 点击"退出登录"切换账号

## 注意事项
- 当前为纯前端实现，密码明文存储在代码中
- 建议后续添加后端验证和加密存储
- 不同设备/浏览器之间的数据不互通
